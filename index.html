安装 HACS

方法1（代理走不了行不通）

  • 文档
  • 命令说明
    • wget -O - https://get.hacs.xyz | bash -
    • -O的含义是把后面网址下载后，改成一个指定的名称，如果后面没有跟着一个名字，而是“-”，则表示将下载后的内容输出到标准输出，也就是输出到屏幕上。
    • 然后通过|，传递给bash执行
    • 减号"-“就是代表标准输出/标准输入, 视命令而定. “-“代替stdin和stdout的用法,stdin就是标准输入,stdout就是标准。 http://http://raw.staticdn.net/hacs/get/main/get GitHub raw 域名并非 http://github.com 而是 http://raw.githubusercontent.com，上方的 GitHub 加速如果不能加速这个域名，那么可以使用 Static CDN 提供的反代服务。

将 http://raw.githubusercontent.com 替换为 http://raw.staticdn.net 即可加速。 —脚本内需要外网不可行

# proxy 必须开启全局代理 https://github.com/Qingquan-Li/blog/issues/131
proxy () {
  export http_proxy=http://127.0.0.1:8024
  export https_proxy=http://127.0.0.1:8024
  # 如果代理需要身份验证，请添加以下行：
  # export HTTP_PROXY_USER=username
  # export HTTP_PROXY_PASS=password
  # 如果你使用 SOCKS 代理，请添加以下行：
  # export ALL_PROXY=socks5://your_proxy_address:port
  echo "HTTP Proxy on, pls run 'curl cip.cc / ip.gs / ifconfig.co/json' to verify."
}

# noproxy
noproxy () {
  unset http_proxy
  unset https_proxy
#  unset all_proxy
  echo "HTTP Proxy off"
}
